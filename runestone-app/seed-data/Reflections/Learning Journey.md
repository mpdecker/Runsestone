# My Learning Journey

Documenting my path through computer science, from databases to AI.

## Where I Started

I began with basic SQL and [[PostgreSQL]] — learning about indexes, queries, and schema design. The relational model was my mental framework for organizing data.

## The Shift

Discovering [[Knowledge Graphs]] changed my thinking. Instead of tables and joins, I started seeing data as nodes and relationships. [[Neo4j]] made this tangible with Cypher queries.

## Current Focus

Now I'm exploring the intersection of:
1. **[[Vector Databases]]** — for semantic understanding
2. **[[Ollama]]** — for local AI capabilities
3. **[[Graph Theory]]** — for the mathematical foundations

## What I've Built

Runestone is the culmination — combining everything into a personal knowledge tool. The [[Architecture]] reflects the lessons learned about balancing different database paradigms.

## What's Next

The [[Ideas]] list keeps growing. Some priorities:
- Better [[Semantic Search]] with hybrid reranking
- Plugin system for community extensions
- Mobile version for capture on the go

> [!tip] Reflection
> The best learning happens when you build something. Theory from [[Graph Theory]] becomes concrete when you're implementing a graph visualization.

## Reading List

- Graph Databases (O'Reilly) — practical [[Neo4j]] guide
- Designing Data-Intensive Applications — covers [[PostgreSQL]] internals
- Speech and Language Processing — background for [[Semantic Search]]
